## **Артиллерийская Битва - Сетевая игра**

**Краткая концепция:**

2-4 игрока заходят в одно из лобби. Когда все игроки нажимают кнопку 
"готов", случайно генерируется 2d карта, на которой находится поле с 
небольшими холмами, на этом поле случайно расставляются артиллерийские 
пушки по одной для каждого игрока. У каждой артиллерийской пушки есть 
по 100 очков здоровья. Игроки могут стрелять из своих пушек, 
выбирая угол и силу выстрела. Игра пошаговая и каждый игрок может 
стрелять только в свой ход. Попадания по артиллерийской пушке другого
игрока сносит его пушке здоровье. Когда здоровье артилеррийской пушки 
игрока падает до нуля, игрок выбывает. Цель игры - уничтожить пушки 
всех остальных игроков. Побеждает последний оставшийся игрок.

При заходе в игру игрок видит окно, в котором можно ввести свой ник и можно выбрать комнату, в которую зайти. При заходе в комнату игроку показывается список из 2-4 игроков и кнопка "готов". После того как все игроки нажимают кнопку "готов" и игра начинается, игроку показывается экран с игрой, на котором карта и пушки игроков. Когда игра заканчивается, игроку показывают результаты игры и есть кнопка "вернуться к выбору лобби"

Требования
1) Многопоточность для сетевого взаимодействия обязательна. Это нужно для разграничения работы интерфейса и сети: чтобы интерфейс не “подвисал” на время приёма или отправки сообщений и наоборот. Важно понимать, как используются потоки.
2) Необходима определенная архитектура сетевого взаимодействия. Она подразумевает взаимодействие 2 и более клиентов. Это взаимодействие client-server-client. Запускается один сервер, к нему подключаются N клиентов отдельными терминалами/устройствами. 
У сервера нет GUI.
3) В игре должен использоваться свой протокол передачи данных через сокеты, содержащий минимум 3 типа сообщений.
Вы можете реализовать протокол как словарь, который сериализуется (через pickle) и передается через сокеты, а потом обратно десериализуется (также через pickle). 
4) Сервер или клиент не должен падать/зависать, если кто-то отключился аварийно
5) Использовать библиотеки и фреймворки, которые полностью решают какую-то задачу логики нельзя (select, pygame, asyncio), но можно использовать компоненты для интерфейса (PyQt5,6, pyside) или библиотеки для работы с данными (pickle, marshmallow, numpy, scipy, pandas…).
